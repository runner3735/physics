  • Components will be automatically set to lower case. This will help to create a unified style for the site, and avoid duplicates that differ only by case. This will also make searches easier from a coding and performance perspective.
  • Components should be singular (e.g., mass, 1 kg not 5 1kg masses. If a particular demo will require a particular number of a given component, that should be specified in the demo description.
  • Components should be specific (e.g., beaker, 3 liter not beaker.
  • It is preferred to specify a component as beaker, 3 liter and not 3 liter beaker. This way, similar components (e.g., beakers) will appear together in the alphabetical list.
  • In the near future, components will have pictures associated with them. This is another reason for them to be singular and specific.
